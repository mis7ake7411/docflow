# USER 私有資料夾樹規格（設計）

日期：2026-03-31

## 目標

讓 `USER` 在檔案管理頁只看到自己建立的資料夾樹，並確保資料夾排序、建立、移動、刪除與文件掛載都只作用在自己的資料夾集合內，避免因共享資料夾樹造成排序語意混亂與權限繞過。

## 使用者價值

- `USER` 可在清楚、穩定的個人資料夾樹中管理自己的文件。
- `USER` 調整資料夾排序時，不會再受到其他使用者資料夾干擾。
- `USER` 不會誤以為自己能操作或掛載到其他人的資料夾。

## 範圍

涵蓋以下對象與情境：
- 對象：`USER` 角色的資料夾樹、資料夾操作、文件掛載資料夾
- 畫面：檔案管理頁左側資料夾樹、資料夾建立/編輯表單、文件列表與文件建立/編輯流程
- 操作：查看資料夾樹、建立資料夾、編輯資料夾、刪除資料夾、同層排序、建立文件、更新文件

## 非目標

- 不新增 `private/public` 通用可視範圍模型
- 不重新設計共享資料夾或個人化排序機制
- 不變更文件分享的授權模型
- 不改變 `ADMIN` / `MANAGER` 的既有可視範圍與高權限行為，除非另有規格明確要求

## 角色規則

### USER

- 只能看到自己建立且未刪除的資料夾
- 只能在自己的資料夾樹中建立、編輯、刪除、移動與排序資料夾
- 建立或更新文件時，只能掛載到自己可見的資料夾

### ADMIN / MANAGER

- 維持既有高權限行為
- 本規格不主動收斂其可視範圍與操作範圍

## 功能規格

### 1) USER 資料夾樹顯示

- `USER` 進入檔案管理頁時，資料夾樹僅顯示自己建立的資料夾
- 他人建立的資料夾不可出現在 `USER` 的資料夾樹中
- 若 `USER` 沒有任何資料夾，畫面應顯示可理解的空狀態，且仍可建立第一個資料夾

### 2) USER 資料夾建立與父節點選擇

- `USER` 建立資料夾時，只能選擇自己的資料夾作為父節點
- 若未指定父節點，系統應建立為該 `USER` 的根資料夾
- `USER` 不得將資料夾建立在他人資料夾底下

### 3) USER 資料夾編輯、移動與刪除

- `USER` 只能編輯自己建立的資料夾
- `USER` 變更資料夾父節點時，只能移到自己的資料夾底下
- `USER` 只能刪除自己建立的資料夾

### 4) USER 資料夾排序

- `USER` 的排序範圍為自己可見資料夾樹中的同層資料夾
- `USER` 調整排序時，只能影響自己的同層資料夾順序
- 排序後，其他使用者的資料夾順序不得因此改變

### 5) USER 文件掛載資料夾

- `USER` 建立文件時，只能掛載到自己的資料夾
- `USER` 更新文件所屬資料夾時，只能改到自己的資料夾
- 若指定的資料夾不屬於目前 `USER` 可見範圍，系統應拒絕該操作

### 6) 前端狀態一致性

- 若目前選取的資料夾在重新載入後不再存在於 `USER` 的可見資料夾樹中，畫面應自動回到「全部文件」狀態
- 畫面不得殘留不存在的資料夾名稱或編號作為目前篩選條件

## 不變項

- 文件分享、共享文件列表、文件詳情中的分享授權規則維持不變
- 文件的檢視、下載與分享權限不因本規格而改變
- `ADMIN` / `MANAGER` 的既有高權限行為不因本規格而退化

## 成功準則

1. `USER A` 看不到 `USER B` 建立的任何資料夾。
2. `USER A` 無法將資料夾建立或移動到 `USER B` 的資料夾底下。
3. `USER A` 無法編輯或刪除 `USER B` 建立的資料夾。
4. `USER A` 建立或更新文件時，無法指定 `USER B` 的資料夾。
5. `USER A` 調整資料夾排序後，只影響自己可見資料夾樹中的同層順序。
6. `USER` 在沒有任何資料夾時，仍可成功建立第一個根資料夾。
7. `ADMIN` / `MANAGER` 的既有高權限使用情境不受本規格影響。
8. 文件分享、共享文件列表與文件詳情的既有授權行為不受本規格影響。

## 驗收情境

### 情境 1：USER 只看見自己的資料夾

- Given 系統中同時存在 `USER A` 與 `USER B` 的資料夾
- When `USER A` 開啟檔案管理頁
- Then `USER A` 只能看見自己建立的資料夾

### 情境 2：USER 不能將資料夾掛到他人父節點

- Given `USER B` 已建立一個資料夾
- When `USER A` 嘗試在建立或編輯資料夾時指定該資料夾為父節點
- Then 系統拒絕操作，且資料不變更

### 情境 3：USER 排序只影響自己的同層資料夾

- Given `USER A` 在同一層有多個自己的資料夾
- When `USER A` 調整該層排序
- Then 只有 `USER A` 該層資料夾順序改變

### 情境 4：USER 不能把文件掛到他人資料夾

- Given `USER B` 已建立一個資料夾
- When `USER A` 建立或更新文件並指定該資料夾
- Then 系統拒絕操作，且文件資料不變更

### 情境 5：選取失效時自動回退

- Given `USER` 目前畫面選取了一個資料夾
- And 該資料夾在重新載入後不再屬於目前 `USER` 的可見資料夾樹
- When 畫面重新整理資料夾樹
- Then 畫面自動回到全部文件狀態，且不顯示失效資料夾識別

## 假設

- 本規格以 `USER` 角色為主；`ADMIN` / `MANAGER` 維持既有高權限行為
- 若既有資料中存在跨使用者父子關係或文件掛載到他人資料夾的情況，需另行定義資料修正策略；本規格僅要求新行為不得再產生此類資料
