# 文件/資料夾權限 UI 行為規格（設計）

日期：2026-03-23

## 目標

讓使用者在 UI 上清楚得知「哪些文件/資料夾可操作、哪些不可操作」，並在權限不足時得到一致且可理解的提示；同時避免出現「按了沒反應」或「以為功能不存在」的混淆。

## 範圍

涵蓋以下畫面與操作：
- 畫面：資料夾列表、文件列表、文件詳情
- 操作：編輯、上傳、刪除

**術語說明**
- 本規格中的「上傳」指 **對既有文件上傳/更新檔案內容（新版本）**，不是「建立新文件」。
- 「建立新文件」不在本規格範圍。

## 非目標

- 不改變既有後端權限規則
- 不影響文件/資料夾的檢視與下載
- 不新增新的角色或權限模型
- 不規範「建立新文件」的流程與權限

## 權限判斷前提（UI 依據）

UI 需取得以下資訊，以判斷是否可操作：
- `currentUser.id` 與 `currentUser.role`
- 目標資料（文件/資料夾）的 `createdBy`

## 權限規則（UI 判斷邏輯）

- ADMIN / MANAGER：可操作任何文件/資料夾（包含 `createdBy` 為空的資料）
- USER：
  - 若 `createdBy == currentUser.id` → 可操作
  - 若 `createdBy != currentUser.id` 或 `createdBy` 為空 → 不可操作

## 全域 UI 行為原則

1. **無權限時仍顯示按鈕，但為不可操作狀態**
   - 避免使用者以為功能不存在
2. **無權限時提供明確提示**
   - 滑過或聚焦於不可操作按鈕時顯示提示文案
3. **若仍觸發操作（快捷鍵/例外路徑）**
   - 顯示即時提示訊息：`無權限操作此文件` 或 `無權限操作此資料夾`
   - 不保存、不更新畫面資料

## 畫面別行為規格

### 1) 資料夾列表

**操作：編輯 / 刪除**
- 可操作：按鈕可點
- 不可操作：按鈕禁用
- 提示文案：`僅能修改自己建立的資料夾`
- 觸發禁止操作時提示：`無權限操作此資料夾`

### 2) 文件列表

**操作：編輯 / 上傳 / 刪除**
- 上傳僅針對**既有文件**（檔案更新）
- 可操作：按鈕可點
- 不可操作：按鈕禁用
- 提示文案：`僅能修改自己建立的文件`
- 觸發禁止操作時提示：`無權限操作此文件`

### 3) 文件詳情

**操作：編輯 / 上傳 / 刪除**
- 上傳僅針對**既有文件**（檔案更新）
- 可操作：按鈕可點
- 不可操作：按鈕禁用
- 按鈕區塊下方可選擇顯示輔助說明：`你僅能修改自己建立的文件`
- 觸發禁止操作時提示：`無權限操作此文件`

## 文案一覽

- 按鈕提示（文件）：`僅能修改自己建立的文件`
- 按鈕提示（資料夾）：`僅能修改自己建立的資料夾`
- 禁止操作提示（文件）：`無權限操作此文件`
- 禁止操作提示（資料夾）：`無權限操作此資料夾`

## 驗收準則（可測）

1. USER 檢視他人文件/資料夾時，編輯/上傳/刪除按鈕顯示但不可操作，且提示文案正確。
2. USER 對他人文件/資料夾觸發操作時，畫面顯示「無權限操作此文件/資料夾」，且資料不變更。
3. USER 對自己建立的文件/資料夾可完成編輯/上傳/刪除。
4. ADMIN/MANAGER 可操作任何文件/資料夾（包含 `createdBy` 為空的資料）。
5. 文件/資料夾的檢視與下載行為不受影響。
