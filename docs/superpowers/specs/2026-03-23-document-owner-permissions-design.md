# 文件管理權限：只能編輯自己文件（設計）

日期：2026-03-23

## 目標

讓一般使用者（USER）只能編輯/上傳/刪除自己建立的文件；管理員（ADMIN）與主管（MANAGER）具備完整文件操作權限。查看與下載不受限制。

## 範圍

包含下列行為的權限限制：
- 文件更新
- 文件上傳
- 文件刪除

不影響：
- 文件列表/詳情檢視
- 文件下載

## 非目標

- 不限制 USER 查看/下載他人文件
- 不改變現有路由角色保護規則
- 不新增角色或新的權限模型

## 權限規則

- USER：
  - 可檢視/下載任何文件
  - 只能編輯/上傳/刪除自己建立的文件
- ADMIN / MANAGER：
  - 可編輯/上傳/刪除任意文件

## 資料流程

1. 進入文件「更新/上傳/刪除」服務流程
2. 取得 currentUser 與 document.createdBy
3. 權限判斷：
   - 若 currentUser.role ∈ {ADMIN, MANAGER} → 允許
   - 若 currentUser.role == USER 且 document.createdBy.id == currentUser.id → 允許
   - 其他情況 → 拒絕

## 錯誤回應

- 權限不足時回傳 403 Forbidden
- 錯誤訊息：`無權限操作此文件`

## 測試策略

- 新增單元測試驗證：
  - USER 對他人文件進行更新/上傳/刪除時返回 403
  - USER 對自己文件進行更新/上傳/刪除時成功
  - ADMIN/MANAGER 對他人文件進行更新/上傳/刪除時成功

## 驗收準則

- USER 嘗試編輯/上傳/刪除他人文件，API 回傳 403 且訊息正確
- USER 編輯/上傳/刪除自己文件成功
- ADMIN/MANAGER 可編輯/上傳/刪除任意文件
- 文件查看/下載行為不受影響
