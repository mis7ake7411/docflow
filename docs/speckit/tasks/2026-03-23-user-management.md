# 使用者管理與新增使用者（任務）

## 任務拆分（對應使用者故事）
1. 故事 US-01：訪客註冊並登入
   任務：新增註冊入口與註冊流程；完成後可登入。
   驗收：依規格「驗收情境 1」。
   成功準則對應：1。

2. 故事 US-02：管理員或 `MANAGER` 建立使用者
   任務：建立管理端新增使用者流程（`ADMIN` 與 `MANAGER` 可用），可設定角色/狀態並產生臨時密碼顯示。
   驗收：依規格「驗收情境 2」。
   成功準則對應：2、4。

3. 故事 US-03：管理端列表與搜尋
   任務：提供使用者列表與單一關鍵字搜尋（username/email），顯示指定欄位。
   驗收：依規格「驗收情境 4」。
   成功準則對應：5。

4. 故事 US-04：管理端更新角色/狀態與啟用停用
   任務：提供更新角色/狀態與啟用/停用能力。
   驗收：依規格「驗收情境 5」。
   成功準則對應：2。

5. 故事 US-05：首次登入改密碼
   任務：建立臨時密碼狀態與強制改密碼流程，未完成不得使用其他功能。
   驗收：依規格「驗收情境 3」。
   成功準則對應：4。

6. 故事 US-06：角色權限邊界
   任務：`MANAGER` 可查看儀表板/文件/資料夾/活動/統計與使用者管理；可新增使用者；不可新增/修改/刪除文件與資料夾。
   驗收：依規格「驗收情境 6」。
   成功準則對應：3。
