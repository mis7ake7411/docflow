# 使用者管理與新增使用者（規格）

## 目標
1. 讓一般使用者可自助註冊並立即使用系統。
2. 讓管理端可建立使用者並設定角色與狀態。
3. 新增 `MANAGER` 角色，具備使用者管理權限與既有資料查看權限。
4. 管理員建立帳號時，系統產生臨時密碼，首次登入必須改密碼。

## 成功準則（可量測、可驗證）
1. 未登入者可在登入頁進入註冊，完成後可成功登入。
2. 管理端可新增使用者並指定角色/狀態，新增後在列表可查到。
3. `MANAGER` 可查看儀表板、文件/資料夾、活動紀錄、統計，並可進入使用者管理；不可新增/修改/刪除文件與資料夾。
4. 使用臨時密碼登入的帳號，首次登入必須完成改密碼後才能使用其他功能。
5. 使用者列表支援單一關鍵字搜尋（username/email），顯示欄位為帳號、Email、角色、狀態、建立時間。

## 使用者故事（MVP，皆可獨立驗收）
1. 以未登入訪客身分，我可以在登入頁看到註冊入口並建立帳號，完成後可登入使用系統。
2. 以管理員身分，我可以建立新使用者並設定角色與狀態，建立後可在列表中查到該使用者。
3. 以管理員或 `MANAGER` 身分，我可以查看使用者列表並用單一關鍵字搜尋（username/email）。
4. 以管理員或 `MANAGER` 身分，我可以更新使用者的角色或狀態，並執行啟用/停用。
5. 以臨時密碼登入的新使用者身分，我必須先完成改密碼才可使用其他功能。

## 功能範圍
包含：
- 自助註冊：登入頁提供入口；註冊預設 `USER + ACTIVE`。
- 使用者管理：列表/搜尋、新增、編輯（角色/狀態）、啟用/停用。
- 角色新增：`MANAGER` 具備使用者管理權限與既有資料查看權限。
- 臨時密碼：管理員建立帳號時產生；首次登入強制改密碼。

不包含：
- 郵件邀請/重設密碼流程。
- 使用者刪除功能。
- 自助重設密碼流程。

## 驗收情境
1. 訪客可完成註冊並登入；註冊後角色為 `USER`、狀態為 `ACTIVE`。
2. 管理端建立使用者時，系統會產生臨時密碼並顯示；新使用者可用該密碼登入。
3. 使用臨時密碼登入的帳號，若未改密碼，系統不得允許進入其他功能。
4. 管理端可在列表搜尋（username/email），結果可見帳號、Email、角色、狀態、建立時間。
5. 管理端可更新使用者角色或狀態，更新後列表立即反映。
6. `MANAGER` 可瀏覽儀表板、文件/資料夾、活動紀錄、統計，但不能執行新增/修改/刪除。

## 例外處理
- 註冊或新增使用者時，若帳號或 Email 重複，需明確提示失敗原因。
- 權限不足者不可見使用者管理入口，若以非授權方式進入應被拒絕。
- 停用帳號不可登入或被拒絕存取受保護功能。
- 改密碼流程需驗證新密碼格式（僅需明確的最小規則即可驗收）。

## 釐清事項（上限 3）
- 無；依下列假設執行。

## 假設與預設
- 方案採用：自助註冊 + 管理員建立 + 新增 `MANAGER`。
- 管理員建立時：產生臨時密碼，首次登入強制改密碼。
- `MANAGER` 權限：可使用使用者管理，且可查看儀表板/文件/資料夾/活動/統計，但不可新增/修改/刪除文件與資料夾。
- 使用者列表欄位：帳號、Email、角色、狀態、建立時間；搜尋為單一關鍵字（username/email）。
